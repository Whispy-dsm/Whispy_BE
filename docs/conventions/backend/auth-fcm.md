# Backend Auth and FCM Convention

## 인증 규칙

- 로그인 요청에는 현재 기기의 `fcmToken`을 포함합니다.
- Access token과 refresh token의 발급, 저장, 무효화는 로그인/로그아웃 흐름에서 함께 관리합니다.
- 로그아웃은 현재 세션 또는 현재 기기 범위를 명확히 구분해서 처리합니다.
- 토큰 재발급 흐름은 refresh token 저장 구조와 만료 정책을 함께 검증합니다.

## FCM 규칙

- 기존 토큰과 다른 토큰으로 로그인하면 이전 기기에 로그아웃 알림을 보냅니다.
- 새 토큰 저장 후 현재 사용자 기준으로 토픽을 재등록합니다.
- 로그인 이후 FCM 토큰 변경은 별도 API로 처리합니다.
- 회원가입 시 `fcmToken`이 있으면 기본 토픽 초기화를 수행합니다.
- topic preference는 계정 단위 정책으로 관리하고, 실제 전송은 활성 기기 전체에 반영되는지 확인합니다.

## 완료 기준

- 같은 계정의 기기 전환 시 이전 기기와 새 기기 상태가 충돌하지 않습니다.
- FCM 토큰 저장, 토픽 재등록, 로그아웃 알림은 실패 시 원인을 추적할 수 있게 예외와 로그를 남깁니다.
- 인증/FCM 변경은 로그인, 로그아웃, 토큰 갱신, 토픽 구독 테스트 범위를 함께 검토합니다.
