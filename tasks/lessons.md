# Lessons

- 2026-03-27: 리뷰 피드백을 받으면 먼저 저장소 컨벤션(패키지 구조, 테스트 위치, 포트 설계, enum 책임)을 확인하고 그 규칙에 맞춰 수정한다.
- 2026-03-30: 사용자가 리뷰를 요청한 경우, 분석 단계와 수정 단계를 분리한다. 승인 전에는 코드 변경을 시작하지 않는다.
- 2026-03-30: 사용자가 컨트롤러 무로직 원칙을 주면 그대로 고정한다. 요청/응답 매핑만 남기고 조립 로직은 서비스나 전용 assembler로 분리한다.
- 2026-04-02: 시그니처 전환 중 임시로 넣은 호환용 default 메서드는 호출처가 모두 새 경로로 옮겨가면 바로 제거한다.
- 2026-04-05: 외부 skill을 가져올 때는 임의 요약본을 새로 쓰지 말고 upstream 본문을 기준으로 맞춘다.
- 2026-04-08: 사용자가 말한 "로그"가 코드 로그가 아니라 작업 로그(tasks)일 수 있다. 의도가 애매하면 기존 맥락을 먼저 확인하고, tasks 기록 요청을 코드 변경으로 해석하지 않는다.
- 2026-04-08: 입력 오류를 다룰 때는 직접 원인(클라이언트 값 불일치)과 서버 책임(예외를 400으로 내릴지 500으로 새는지)을 분리해서 판단한다.
- 2026-04-09: 터미널 출력 인코딩이 깨져 보여도 파일 자체 인코딩 문제라고 단정하지 않는다. 실제 파일 상태와 사용자 맥락을 먼저 확인하고, 표현 언어(한글/영문)는 임의로 바꾸지 않는다.
- 2026-04-09: 사용자가 Javadoc 언어 일관성을 지적하면 이번 변경분 전체를 다시 훑어서 같은 범위의 주석 언어를 통일한다. 새 파일만 고치고 끝내지 않는다.
- 2026-04-12: 단순한 Git 병합/푸시 작업이라도 이 저장소에서는 작업 시작 전에 `tasks/todo.md`에 최소 체크리스트를 남긴다. 작업 후에는 결과, 검증, 미실행 항목을 같은 로그에 기록한다.
- 2026-04-13: 이 작업공간에서 `$team` / `omx team`을 실행할 때는 권한 상승으로 실행한다. sandbox 안에서는 Node `child_process`가 `tmux` 실행을 `EPERM`으로 막아 `omx team`이 tmux를 찾지 못한 것처럼 실패할 수 있다.
- 2026-04-13: 기능명세서를 작성할 때는 여러 도메인을 `Query` 같은 임시 분류로 묶지 않는다. 기능 단위는 PRD 수준으로 쪼개고, 분류는 `SLEEP`, `STATISTICS`, `NOTIFICATION`처럼 실제 도메인/화면 책임에 맞춘다.
- 2026-04-13: PRD 문서는 항목 나열로 끝내지 않는다. 관련 기능은 `인증/계정`, `세션`, `음악 경험`, `알림/토픽`, `결제/구독`, `운영`처럼 상위 도메인 묶음으로 다시 볼 수 있게 정리한다.
- 2026-04-13: 로그인/회원가입 문서를 쓸 때는 FCM 토큰과 기기 전환 흐름을 빠뜨리지 않는다. 현재 저장소는 로그인 시 `fcmToken` 필수, 기존 기기 로그아웃 알림, 토픽 재등록이 실제 동작 규칙이다.
- 2026-04-13: stash가 현재 작업의 source of truth라면 수동 재작성으로 우회하지 않는다. 먼저 stash commit과 working tree를 대조하고, tracked/untracked 파일을 각각 stash에서 정확히 복구한 뒤 이어서 검증한다.
- 2026-04-13: 이 저장소에서 이슈가 있는 작업을 커밋할 때는 첫 줄부터 `타입 ( #이슈번호 ) : 설명` 형식을 지킨다. 나중에 amend로 고치지 않도록 커밋 전에 제목을 다시 확인한다.
- 2026-04-13: Windows PowerShell에서 한글 커밋 메시지를 파이프로 바로 `git commit -F -`에 넘기면 `?`로 깨질 수 있다. 한글 커밋은 UTF-8 파일로 메시지를 쓰고 amend/commit 후 `git log --pretty=%B -1`로 실제 저장값을 검증한다.
- 2026-04-13: 백엔드 release note는 버전 규칙을 먼저 적용한다. `v1.0.0` 같은 major 초기 릴리즈 페이지에 후속 fix/refactor 내용을 덧붙이지 말고, 큰 수정은 `v1.1.0`, 유지보수성 리팩터링은 patch/minor 규칙에 맞는 새 페이지로 분리한다.
- 2026-04-14: `$team`이 특정 repo에서 실패해도 OMX 전체 실패로 단정하지 않는다. `C:\Users\user` 같은 trusted/non-repo cwd와 repo worktree cwd는 Codex trust 설정, worktree mode, shared `.omx/state` 쓰기 권한, PowerShell JSON quoting 조건이 다르므로 먼저 비교한다.
- 2026-04-14: `$team` worker pane이 아예 task protocol에 못 들어가면 먼저 worker CLI 인증 상태를 확인한다. Codex는 로그인 화면, Claude는 `Please run /login` / 401 인증 오류로 멈출 수 있고, 이 경우 ACK/claim 실패를 runtime 버그로만 보면 안 된다.
- 2026-04-14: OMX team worker launch args는 `config.toml`의 `[env]`만으로는 런타임 shell에 그대로 반영되지 않을 수 있다. worker bypass 플래그가 꼭 필요하면 `OMX_TEAM_WORKER_LAUNCH_ARGS=... omx team ...`처럼 실행 시점 shell env로도 넘겨서 검증한다.
- 2026-04-16: 이 저장소에서 이슈 기반 작업을 커밋할 때는 한 이슈라도 변경 책임을 가능한 범위에서 쪼갠다. 최소한 `test`, `controller/dto/doc`, `service/port/persistence`는 분리 검토하고, 한 번에 뭉치지 않는다.
- 2026-04-20: 응답에 표시용 상대 시간이 필요해 보여도 백엔드는 먼저 원천 데이터(`createdAt` 등)를 내려주는 단순한 계약을 검토한다. 클라이언트 표시 정책이면 백엔드에서 `timeAgo` 같은 문구를 만들지 않는다.
