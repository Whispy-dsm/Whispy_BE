# Lessons

- 2026-03-27: 리뷰 피드백이 들어오면 먼저 저장소 컨벤션(패키지 구조, 테스트 위치, 포트 설계, enum 책임)을 재검색하고 맞춘 뒤 계속 구현한다. 특히 external adapter는 `adapter/out/external`, test-only 어댑터는 test source로 분리하고, enum에는 매핑 로직을 넣지 않는다.
- 2026-03-30: 사용자가 리뷰를 요청했거나 분석 단계 승인만 준 상태에서는 구현 승인으로 해석하지 않는다. 리뷰 결과와 영향 범위를 먼저 고정하고, 사용자가 명시적으로 수정 승인하기 전에는 애플리케이션 코드 변경을 시작하지 않는다.
- 2026-03-30: 사용자가 "controller에 로직 없어야 한다"는 구조 원칙을 주면 그대로 고정한다. 컨트롤러에는 요청/응답 매핑만 두고, 헤더 계산·미디어타입 결정·캐시 정책 조립 같은 로직은 서비스/전용 assembler로 분리한다.
